# 읽기 훈련 앱(Reading Trainer) 설계 명세서

> 대상: 만 10세 전후 아동(직접 조작) + 옆에서 돕는 부모. 여러 아이를 프로필로 구분.
> 목적: 문해력 향상 + 좌→우 시선 처리(안구 운동) 훈련.
> 환경: Windows 데스크톱 / Electron / 단일 .exe 설치본.
> 본 문서는 `first_step.md`(CO-STAR 설계 프롬프트)의 실행 산출물이며, `docs/plan/2026-06-28-reading-trainer-plan.md`의 **결정 기록(D1~D10)**을 전제로 한다.
>
> ✅ 진행 상태: **본문 1~8장 전체 작성 완료.** (Phase 0 산출물 완성 — 휴먼 최종 승인 대기.)

## [1단계] 사고 과정 (A → B → C)

### A. 핵심 설계 난제 정리

1. **스윕 바 속도-거리 계산** — 기준 1.0배 = 400 CPM. 공백 제외 카운트라 "시간 축(글자 수 기반)"과 "픽셀 축(화면 위치)"이 서로 다른 단위다. 1글자당 시간(ms/char)과 1프레임당 이동량(px/frame)을 어떻게 연결할 것인가.
2. **문장 끝 0.3초 멈춤** — 마침표·물음표·느낌표 등 종결부에서 바를 300ms 멈췄다가 이어야 한다. 이 정지(hold)를 시간 타임라인에 자연스럽게 끼우면서 위치(x)는 고정시키는 방법.
3. **공백 처리** — 공백은 시간 0(카운트 제외)이지만 화면에는 폭이 있다. 시간 0으로 두면 바가 공백 구간을 순간이동(텔레포트)한다. 부드럽게 이으려면 공백 폭을 어디에 흡수시킬지 결정해야 한다.
4. **3/4/5줄 페이지 분할 + 글자 크기 변경 시 재배치** — 글꼴·글자 크기에 따라 한 줄에 들어가는 글자 수가 달라진다. core는 글자 수로 1차 분할만 하고, 실제 줄바꿈은 Frontend가 픽셀 폭을 측정해 보정해야 한다(글자 크기 −/+ 시 재측정·재빌드).
5. **라이트/다크 테마와 스윕 바 색·투명도** — 라이트=연한 노랑, 다크=앰버(#F0C674), 공통 글자폭 1.5배·투명도 50%. 테마는 프로필별 기억.
6. **속도 4단계(배율↔CPM) 동기화 + 타이머 우선 종료** — 0.5/1.0/1.5/2.0배 = 200/400/600/800 CPM. 속도가 바뀌면 타임라인 전체가 재계산되어야 하고, 타이머가 끝나면 글 중간이어도 즉시 종료해야 한다.
7. **일시정지/재개 상태 보존** — 멈춘 지점(어느 페이지·줄·줄 내 경과 ms), 남은 시간, 진도를 모두 보존하고 재개 시 정확히 그 지점부터 이어야 한다.
8. **5분마다 20초 강제 휴식(스킵 불가) + 명언** — "5분"은 활성 읽기 시간 기준. 휴식 중에는 타이머·스윕 바 정지. 20초 카운트다운을 다 채워야 복귀. 명언은 1000개에서 무작위, 최근 표시 제외(중복 방지).
9. **텍스트 입력 2경로 통일** — ① 붙여넣기 ② .txt 열기. 둘 다 같은 내부 형식(정규화된 `\n` 텍스트)으로 변환해 프로필에 저장하고 목록에서 재선택.
10. **다중 프로필 + SQLite 저장** — 프로필·설정·글·세션·진도·명언·명언 표시 이력을 프로필 단위로 분리 저장. 프로필 삭제 시 연관 데이터 정리(FK CASCADE).
11. **아동 UX·눈 피로·개인정보** — 큰 버튼, 적은 선택지, 강제 휴식, 다크모드. 실명 대신 별명+아바타만 받고 로컬에만 저장.

### B. 각 난제에 대한 설계 결정과 근거 (1~2줄)

1. **속도-거리:** `ms/char = 60000 / (400×배율)`(1.0배=150ms). 픽셀 위치는 별도로 글꼴 측정값(`glyphX[]`)에서 가져온다. → 시간은 글자 수로, 위치는 측정 폭으로 분리해 공백 제외 규칙과 부드러운 이동을 동시에 만족.
2. **문장 끝 멈춤:** 종결부 글자 세그먼트에 `holdMs=300`을 부여하고, 그 hold 구간 동안 `x`를 글자 끝 위치에 고정. → 시간 타임라인에 자연 삽입되고 위치는 안 튐.
3. **공백:** 공백은 시간 0으로 건너뛰되, 그 폭은 **다음 글자 세그먼트의 시작 x로 흡수**(D1 보강). → 텔레포트 없이 부드럽게 이동, 카운트는 공백 제외 유지.
4. **페이지/재배치 (D1):** core `paginate()`는 글자 수 기준 1차 분할 → Frontend가 글꼴 로드 후 픽셀 폭으로 재측정·보정, 글자 크기 −/+ 시 `glyphX` 재측정 후 타임라인 재빌드. → 로직(UI 비의존 TDD)과 정확한 화면 표시 분리.
5. **테마:** core/로직은 색을 모르고, Frontend `theme.ts`가 라이트=연한 노랑·다크=#F0C674, 폭 1.5배·투명도 50% 적용. 프로필별 `settings.theme` 기억.
6. **속도·종료 (D3):** 속도 변경 시 타임라인 재빌드. 세션 시계는 **활성 읽기 시간만** 누적하고 `totalMs` 도달 시 즉시 종료(타이머 우선). 글이 먼저 끝나면 처음부터 반복(D4).
7. **일시정지/재개:** 세션 상태를 {페이지·줄 인덱스, 줄 내 경과 ms, 남은 시간, 진도}로 모델링. 일시정지 = 시계·바 동시 정지, 재개 = 저장된 경과 ms부터 `sweepXAt()` 재개.
8. **휴식·명언 (D6):** `breakScheduler`가 활성 300000ms 누적마다 휴식 트리거, `BREAK_MS=20000`, 스킵 API 없음(스킵 불가). 명언은 `pickQuote(all, recentIds, rng)`로 최근 제외 후 무작위(결정론 테스트 위해 rng 주입).
9. **텍스트 입력:** 붙여넣기·`.txt` 열기 모두 → 줄바꿈 정규화(`\r\n`→`\n`) → 같은 `body` 형식으로 `texts` 테이블에 저장 → StartScreen 목록에서 재선택.
10. **프로필·DB:** 프로필별 `settings/texts/sessions/quote_history` 분리, FK `ON DELETE CASCADE`로 프로필 삭제 시 정리. 명언 본문은 전역 `quotes`, 표시 이력은 프로필별 `quote_history`.
11. **아동 UX·개인정보 (D5·D10):** Pretendard 24pt↑ 기본, 큰 버튼·적은 선택지. 프로필은 **별명+아바타만**(실명 X), 모든 데이터 로컬 SQLite·외부 전송 없음을 화면에 명시.

### C. 설계 방향 요약

- **순수 로직과 UI를 분리한다.** 가장 어려운 산수(속도·페이지·스윕 바·휴식·명언)는 `src/core/`의 Electron/DOM 비의존 순수 모듈로 떼어 값→값으로 TDD하고, Frontend는 글꼴 픽셀 폭을 측정해 core에 주입한다.
- **시간 축과 픽셀 축을 분리해 연결한다.** 시간은 "공백 제외 글자 수 × ms/char(+문장끝 hold)", 위치는 "측정된 glyphX". 이 둘을 `Segment` 타임라인으로 묶어 부드러운 이동·문장끝 멈춤·공백 흡수를 동시에 해결한다.
- **타이머가 최우선 종료 기준**이며 활성 시간만 센다. 일시정지·휴식은 모든 시계를 멈추고 상태를 보존한다.
- **아동 안전을 기본값으로.** 강제 휴식(스킵 불가), 다크모드, 큰 글자, 별명+아바타(실명 비수집), 완전 로컬 저장.

## [2단계] 설계 명세서 본문 — 목차 및 각 장 개요 (1~2줄)

> 아래는 **개요판**이다. "계속" 시 1장부터 약 3000자씩 상세 본문을 이어 쓴다.

**1. 개요 및 학습 목표**
앱의 목적(문해력·좌→우 시선 훈련)과 대상(10세 아동+부모), 핵심 학습 원리(일정 속도 스윕 바를 눈으로 따라가며 끊김 없는 읽기 습관 형성)를 한눈에 정리한다.

**2. 사용자 시나리오**
프로필 선택 → 글 입력/선택 → 옵션 설정(줄수·속도·타이머·테마·글자크기) → 읽기(스윕 바 따라가기) → 5분마다 강제 휴식+명언 → 타이머 종료 → 기록 저장까지의 전체 흐름을 이야기 형식으로 묘사한다.

**3. 화면(UI) 구성**
5개 화면(프로필 / 시작 / 읽기 / 휴식 / 종료) 각각의 구성 요소·버튼·동작을 글로 묘사. 큰 버튼·적은 선택지·별명+아바타(실명 X) 원칙 포함.

**4. 핵심 기능 상세 (4-1 ~ 4-10)**
프로필, 텍스트 입력 2경로 정규화, 페이지 표시(3/4/5줄·Pretendard 24pt↑·−/+), 테마(연노랑/앰버·폭1.5배·투명50%), 타이머(활성시간·우선종료), 속도(배율↔CPM↔ms/char), 스윕 바(타임라인·문장끝 0.3초·공백 흡수·줄/페이지 전환, 의사코드 포함), 일시정지/재개, 휴식+명언(5분·20초·스킵불가·중복방지·데이터구조·예시 5개), 진도·기록 저장을 항목별로 상세화한다.

**5. 데이터 구조 & DB**
앱/세션 런타임 상태 구조 + SQLite 테이블 스키마(`profiles`·`settings`·`texts`·`sessions`·`quotes`·`quote_history`)를 코드블록으로 제시. FK CASCADE·프로필별 분리 저장 명시.

**6. 아동 안전·접근성·눈 피로 배려**
밝기/다크모드, 일시정지, 강제 휴식(스킵 불가), 큰 글자, 적은 선택지, 별명+아바타·완전 로컬 저장(외부 전송 없음) 원칙을 정리한다.

**7. Electron 구성 개요**
main(SQLite·파일 열기·창 관리) / preload(contextBridge 안전 IPC) / renderer(UI+읽기 엔진) 3분리 역할과, better-sqlite3 ABI 재빌드(Phase 1 선검증)·단일 .exe(NSIS) 패키징을 개념 수준으로 설명한다.

**8. 향후 확장 아이디어**
난이도 단계, 프로필별 읽기 리포트·통계 그래프, 명언 즐겨찾기, 글 카테고리/추천, 보호자용 요약 등 후속 확장 후보를 나열한다.

# 설계 명세서 본문

## 1. 개요 및 학습 목표

### 1-1. 이 앱은 무엇인가
이 앱은 만 10세 전후 아이가 **줄글(이어진 문장)을 왼쪽에서 오른쪽으로 시선을 끊지 않고 읽도록 도와주는 읽기 훈련 프로그램**입니다. 화면에는 아이가 읽을 글이 한 번에 3줄·4줄·5줄 중 선택한 만큼 보이고, 그 위로 **반투명한 세로 막대(이하 "스윕 바")**가 일정한 속도로 천천히 왼쪽→오른쪽으로 움직입니다. 아이는 이 막대를 눈으로 따라가며 글을 읽습니다. 막대는 한 줄을 다 지나면 다음 줄의 시작점으로 내려가고, 페이지의 마지막 줄까지 끝나면 다음 페이지로 넘어갑니다.

쉽게 비유하면, **노래방의 가사 강조 막대(따라 부르도록 색이 차오르는 것)**와 비슷합니다. 다만 노래가 아니라 "읽기 속도"에 맞춰 막대가 움직이고, 아이의 눈이 막대를 쫓아가면서 자연스럽게 좌→우 안구 운동을 반복하게 됩니다.

### 1-2. 왜 만드는가 (학습 목표)
- **문해력(글을 읽고 이해하는 힘) 향상:** 일정한 속도로 끝까지 읽는 경험을 반복하면, 중간에 멈칫하거나 같은 줄을 다시 읽는 습관(역행)이 줄어듭니다.
- **시선 처리(안구 운동) 훈련:** 왼쪽→오른쪽으로 시선을 끊김 없이 옮기는 연습. 줄 끝에서 다음 줄로 시선을 정확히 내리는 동작도 함께 익힙니다.
- **집중 지속력:** 정해진 시간(5~20분) 동안, 중간에 강제 휴식을 끼워 눈과 집중력을 보호하면서 꾸준히 읽는 습관을 들입니다.

### 1-3. 무엇을 중요하게 여겼나 (설계 원칙)
1. **아이 눈높이와 안전 우선** — 큰 글자(기본 24pt 이상, 받침이 잘 보이는 Pretendard 고딕), 적은 선택지, 큰 버튼, 다크모드, **5분마다 20초 강제 휴식(건너뛰기 불가)**.
2. **부모도 이해 가능** — 설정 항목을 최소화하고, 화면마다 무엇을 하는 곳인지 분명히 합니다.
3. **개인정보 최소 수집** — **실명을 받지 않고 별명과 아바타(그림)로만** 아이를 구분합니다. 모든 기록은 **이 컴퓨터 안(로컬)에만 저장**되고 인터넷으로 전송되지 않습니다.
4. **여러 아이 지원** — 형제·자매나 친구 여러 명을 **프로필**로 나눠, 각자의 글·설정·기록을 따로 보관합니다.

### 1-4. 성공 기준(이 앱이 잘 만들어졌다고 보는 조건)
- 아이가 도움 없이도 "내 프로필 고르기 → 글 고르기 → 읽기 시작" 까지 스스로 할 수 있다.
- 스윕 바가 설정한 속도(예: 1.0배=분당 400자)에 맞춰 **부드럽게** 움직이고, 문장 끝에서 잠깐 멈춘다.
- 타이머가 끝나면 글 중간이라도 깔끔히 종료되고, 읽은 분량이 기록된다.
- 5분마다 휴식 화면이 뜨고 명언이 보이며, 20초를 다 채워야 다시 읽을 수 있다.

## 2. 사용자 시나리오 (전체 흐름)

아래는 아이(민지, 10세)와 부모가 앱을 한 번 사용하는 과정을 처음부터 끝까지 따라간 이야기입니다.

### 2-1. 프로필 선택
민지가 앱을 켜면 **프로필 화면**이 먼저 나옵니다. 큰 카드 형태로 아이들의 **아바타 그림과 별명**이 보입니다("민지", "동생토끼" 등). 민지는 자기 카드를 눌러 들어갑니다. (처음이라면 부모가 "새 친구 추가"를 눌러 별명을 적고 아바타를 하나 골라 줍니다. 실명은 적지 않습니다.)

### 2-2. 글 입력 또는 선택
프로필을 고르면 **시작 화면**으로 넘어갑니다. 민지(또는 부모)는 세 가지 중 하나로 읽을 글을 정합니다.
- **붙여넣기:** 다른 곳에서 복사한 글을 입력칸에 붙여넣습니다.
- **.txt 파일 열기:** 컴퓨터에 있는 텍스트 파일을 불러옵니다.
- **저장된 글 고르기:** 예전에 넣어 둔 글 목록에서 다시 고릅니다.
새로 넣은 글은 제목과 함께 **민지 프로필에 저장**되어 다음에 또 고를 수 있습니다.

### 2-3. 옵션 설정
같은 시작 화면에서 읽기 조건을 고릅니다. 모두 큰 버튼이고 선택지는 적습니다.
- **한 화면 줄 수:** 3 / 4 / 5줄
- **속도:** 0.5 / 1.0 / 1.5 / 2.0배 (1.0배 = 분당 400자)
- **시간(타이머):** 5 / 10 / 15 / 20분
- **테마:** 라이트(밝게) / 다크(어둡게)
- **글자 크기:** − / + 버튼으로 키우거나 줄임 (기본 24pt↑)
이 설정들은 **민지 프로필에 기억**되어 다음에 자동으로 그대로 적용됩니다.

### 2-4. 읽기
"시작"을 누르면 **읽기 화면**이 뜹니다. 글이 선택한 줄 수만큼 보이고, 반투명 세로 막대가 첫 줄 왼쪽에서 출발해 오른쪽으로 일정하게 움직입니다. 민지는 막대를 눈으로 따라가며 읽습니다. 막대는 **문장 끝(마침표·물음표·느낌표)에서 0.3초 잠깐 멈췄다가** 다시 갑니다. 줄 끝에 닿으면 다음 줄 처음으로 내려가고, 페이지가 끝나면 다음 페이지로 넘어갑니다. 읽다가 힘들면 **일시정지** 버튼으로 막대와 시간을 함께 멈추고, **재개**하면 멈춘 그 자리·남은 시간부터 이어집니다.

### 2-5. 강제 휴식 + 명언
읽기를 시작한 뒤 **활성 읽기 시간이 5분** 쌓이면, **휴식 화면**이 자동으로 뜹니다(막대와 타이머는 멈춤). 화면 가운데에 **공부·배움·성장에 관한 짧은 명언 한 문장**이 크게 보이고, **20초 카운트다운**이 돕니다. 이 휴식은 **건너뛸 수 없고**, 20초를 다 채운 뒤에야 읽기로 돌아갑니다. 명언은 1000개 중에서 무작위로 뽑되 최근에 본 것은 잠시 제외해 잘 겹치지 않습니다.

### 2-6. 종료 + 기록
**선택한 타이머 시간이 끝나면** 글을 다 읽지 못했어도 세션이 종료됩니다(시간 우선). 반대로 글을 끝까지 다 읽었는데 시간이 남으면 **같은 글을 처음부터 다시** 돌립니다. 끝나면 **종료 화면**에 "잘했어요!" 같은 격려와 함께 오늘 읽은 분량·진도가 요약되고, 이 기록이 **민지 프로필에 저장**됩니다.

## 3. 화면(UI) 구성

전체 화면은 **5개**입니다: 프로필 → 시작 → 읽기 → 휴식 → 종료. 공통 원칙은 **큰 글자·큰 버튼·적은 선택지**, 그리고 **현재 무엇을 하는 화면인지 한눈에 보이게** 하는 것입니다. 모든 화면은 라이트/다크 테마를 따릅니다.

### 3-1. 프로필 화면 (Profile Screen)
앱을 켜면 가장 먼저 나오는 화면입니다.
- **구성:** 화면 가운데에 아이들의 **아바타 그림 + 별명**이 적힌 **큰 카드**들이 나란히 놓입니다. 카드를 누르면 그 아이로 들어갑니다.
- **새 친구 추가:** "＋ 새 친구" 카드를 누르면 작은 입력창이 뜹니다. **별명(텍스트) 입력 + 아바타 그림 선택**만 받습니다. (실명·생년월일 등은 받지 않음 — D10.)
- **삭제:** 각 카드 모서리의 작은 "톱니/연필" 아이콘 → "이 친구 지우기". 지우면 그 아이의 글·설정·기록이 함께 삭제된다는 안내를 부모용으로 한 번 더 확인합니다.
- **안내 문구:** 화면 하단에 "모든 기록은 이 컴퓨터에만 저장돼요" 같은 한 줄 안내(개인정보 안심 문구).

### 3-2. 시작 화면 (Start Screen)
프로필을 고르면 나오는, **글을 정하고 옵션을 고르는** 화면입니다. 크게 두 영역으로 나뉩니다.

**(A) 글 정하기 영역**
- **붙여넣기 칸:** 큰 텍스트 입력칸 + "붙여넣기" 버튼. 여기에 글을 넣고 "이 글로 저장" 하면 제목을 묻고 목록에 추가됩니다.
- **.txt 열기 버튼:** 누르면 파일 선택창(.txt만)이 떠서 파일을 불러옵니다. 불러온 글도 목록에 저장됩니다.
- **저장된 글 목록:** 이 프로필이 예전에 넣은 글들이 제목 카드로 보이고, 누르면 선택됩니다.

**(B) 옵션 고르기 영역** (모두 큰 토글/버튼, 현재 선택은 굵게 강조)
- **줄 수:** [3] [4] [5]
- **속도:** [0.5×] [1.0×] [1.5×] [2.0×] (아래 작은 글씨로 "1.0배 = 분당 400자")
- **시간:** [5분] [10분] [15분] [20분]
- **테마:** [밝게 ☀] [어둡게 🌙]
- **글자 크기:** [－] 24pt [＋]
- 하단에 큰 **[읽기 시작]** 버튼. (글이 선택되지 않았으면 버튼이 흐리게 비활성.)
- 모든 옵션은 **선택 즉시 이 프로필 설정으로 저장**되어 다음에 그대로 복원됩니다.

### 3-3. 읽기 화면 (Reading Screen) — 핵심 화면
실제로 글을 읽는 화면입니다. 군더더기를 최소화해 글과 막대에 집중하게 합니다.
- **본문 영역:** 선택한 줄 수(3/4/5줄)만큼 글이 큰 글씨로 보입니다. 한 페이지 분량이며, 다 읽으면 다음 페이지로 바뀝니다.
- **스윕 바:** 본문 위를 덮는 **반투명 세로 막대**. 글자폭의 **1.5배 너비, 투명도 50%**, 색은 테마에 따라 **라이트=연한 노랑 / 다크=앰버(#F0C674)**. 현재 읽는 **한 줄** 위를 왼쪽→오른쪽으로 움직이고, 줄 끝에서 다음 줄 시작으로 내려갑니다. 문장 끝에서 0.3초 멈춥니다.
- **상단 상태 표시:** 남은 시간(타이머), 현재 페이지/전체 페이지. 작고 방해되지 않게.
- **하단 조작 버튼:** 큰 **[일시정지/재개]** 버튼 하나가 중심. (속도·글자크기 미세 조정은 작은 보조 버튼으로 둘 수 있으나, 기본은 단순하게.)
- **일시정지 상태:** 막대와 타이머가 멈추고, 화면에 "잠깐 멈춤" 표시. 재개하면 멈춘 지점·남은 시간부터 이어집니다.

### 3-4. 휴식 화면 (Break Screen)
활성 읽기 5분마다 자동으로 덮이는 화면입니다.
- **구성:** 차분한 배경(테마색) 위에 **명언 한 문장**을 화면 가운데 크게 보여 줍니다.
- **카운트다운:** "20초 후에 다시 시작해요" → 20…19…18… 숫자가 줄어듭니다.
- **스킵 불가:** 건너뛰기 버튼이 **없습니다.** 20초를 다 채워야 자동으로 읽기로 복귀합니다. (눈·집중력 보호 — 강제 휴식.)
- 읽던 위치와 남은 시간은 그대로 보존되어, 휴식이 끝나면 멈췄던 지점부터 이어집니다.

### 3-5. 종료 화면 (End Screen)
타이머 시간이 끝나면 나오는 마무리 화면입니다.
- **격려 메시지:** "오늘도 끝까지 읽었어요! 잘했어요 👏" 같은 큰 칭찬 문구 + 알림(소리/표시).
- **요약:** 오늘 읽은 분량(글자 수/페이지), 읽은 시간, 사용한 속도 등을 큰 글씨로 간단히 보여 줍니다.
- **버튼:** [다시 읽기] (같은 글·설정으로 새 세션) / [시작 화면으로] / [프로필 바꾸기].
- 이 세션 기록은 자동으로 프로필에 저장됩니다.

## 4. 핵심 기능 상세

> 이 장은 개발자가 그대로 구현에 들어갈 수 있도록, 기능별 규칙·계산식·상태 처리를 구체적으로 적는다. 가장 어려운 로직(4-6 속도, 4-7 스윕 바)은 **Electron/화면에 의존하지 않는 순수 모듈(`src/core/`)**로 만들어 값→값으로 테스트한다.

### 4-1. 사용자 프로필 — 등록 / 선택 / 삭제, 데이터 분리

- **데이터 모델:** 프로필 1개 = `profiles` 한 행(별명 `name`, 아바타 `avatar`, 생성일). 실명은 받지 않는다(D10).
- **분리 저장:** 한 프로필에 **설정(`settings`)·글(`texts`)·세션 기록(`sessions`)·명언 표시 이력(`quote_history`)**이 1:N으로 매달린다. 모든 조회는 항상 `profile_id`로 필터한다.
- **선택:** 프로필 선택 시 그 프로필의 `settings`를 읽어 화면 옵션(줄수·속도·시간·테마·글자크기)을 복원한다. 설정이 없으면 기본값(5줄·1.0배·10분·라이트·24pt)을 만든다.
- **삭제:** `profiles` 행을 지우면 외래키 `ON DELETE CASCADE`로 그 프로필의 `settings/texts/sessions/quote_history`가 함께 삭제된다(전역 `quotes` 본문은 유지). 삭제 전 부모용 확인 1회.

### 4-2. 텍스트 입력 — 2경로 → 같은 내부 형식으로 정규화

두 입력 경로의 결과를 **동일한 내부 형식**으로 맞춘 뒤 저장한다.
- **경로 ①(붙여넣기):** 입력칸의 문자열을 그대로 받는다.
- **경로 ②(.txt 열기):** main 프로세스의 파일 대화상자(`dialog.showOpenDialog`, `.txt` 필터)로 파일을 읽어 문자열로 받는다. 인코딩은 UTF-8 기본, 필요 시 감지.
- **정규화 규칙(공통):** ① 줄바꿈 통일(`\r\n`·`\r` → `\n`), ② 끝쪽 불필요한 공백 줄 정리, ③ 탭 → 공백. 결과 문자열이 `texts.body`가 된다.
- **저장·재선택:** 제목(`title`) + 본문(`body`)을 `texts`에 프로필별 저장 → 시작 화면 목록에서 다시 고른다. 같은 글을 또 넣으면 새 행(중복 허용)으로 두되, 제목으로 구분.

### 4-3. 페이지 표시 — 3/4/5줄, 줄 분할, 폰트·글자 크기

- **폰트:** Pretendard(받침이 또렷한 고딕). 앱에 동봉(OFL). 기본 24pt 이상, 화면 −/+ 버튼으로 확대·축소(프로필별 기억).
- **2단계 분할(D1):**
  1. **1차(core, 글자수 기준):** `paginate(text, linesPerPage, maxCharsPerLine)`가 어절(공백) 경계로 줄을 나누고 `linesPerPage`마다 페이지를 끊는다. UI 없이 테스트 가능.
  2. **2차(Frontend, 픽셀 보정):** 실제 Pretendard·현재 글자 크기로 각 줄의 폭을 측정해, 화면을 넘치면 줄을 더 쪼개고 모자라면 합쳐 최종 레이아웃을 만든다.
- **글자 크기 변경 시 재배치:** −/+로 크기가 바뀌면 **글자별 픽셀 위치(`glyphX`)를 다시 측정 → 페이지 재분할 → 스윕 바 타임라인 재빌드**. 즉, 크기 변경은 "다시 그리기"를 유발한다.
- **페이지 전환:** 한 페이지의 마지막 줄에서 스윕 바가 끝나면 다음 페이지로 넘어간다(4-7과 연동).

### 4-4. 테마 — 라이트 / 다크, 스윕 바 색·폭·투명도

- **라이트 모드:** 밝은 배경/진한 글자. **스윕 바 = 연한 노랑.**
- **다크 모드:** 어두운 배경/밝은 글자. **스윕 바 = 따뜻한 앰버(#F0C674 계열).**
- **공통 스윕 바 속성:** **글자폭의 1.5배 너비, 투명도 50%**(반투명이라 글자가 비쳐 보임).
- **저장·즉시 반영:** 테마는 `settings.theme`에 프로필별 저장. 전환 시 배경·글자색·스윕 바 색이 즉시 바뀐다. 색·투명도 같은 표현은 Frontend `theme.ts`가 전담하고, core 로직은 색을 모른다(관심사 분리).

### 4-5. 타이머(5/10/15/20분) — 동작, 종료 알림, 타이머 우선 종료

- **단위:** 5분 단위 4개(5/10/15/20분). 내부적으로 `totalMs = 분×60000`.
- **활성 시간 기준(D3):** 타이머는 **실제로 읽는 동안만** 줄어든다. **일시정지·휴식 중에는 멈춘다**(벽시계가 아니라 활성 시간). 이래야 "20분 읽기"가 진짜 20분의 읽기 훈련이 된다.
- **타이머 우선 종료:** 활성 시간이 `totalMs`에 도달하면 **글 중간이어도 즉시 세션을 종료**하고 종료 화면으로 간다.
- **글이 먼저 끝나면(D4):** 페이지를 끝까지 읽었는데 시간이 남으면 **같은 글을 처음부터 다시** 시작해 시간이 찰 때까지 반복한다.
- **종료 알림:** 종료 시 알림(소리/표시)과 함께 읽은 분량을 요약하고 세션을 저장(4-10).

---

### 4-6. 속도(0.5/1.0/1.5/2.0배) — 배율 → CPM → 1글자당 시간

- **기준:** 1.0배 = **400 CPM**(분당 400자). 배율별: 0.5배=200, 1.5배=600, 2.0배=800 CPM.
- **글자 수 카운트 규칙:** **공백(띄어쓰기·줄바꿈) 제외**, 한글 글자와 문장부호만 1자로 센다.
- **계산식:**
  - `cpm = 400 × 배율`
  - `1글자당 시간(ms) = 60000 / cpm` → 1.0배 = 150ms, 0.5배 = 300ms, 1.5배 = 100ms, 2.0배 = 75ms.

```ts
// core/cpm.ts — UI/Electron 비의존 순수 함수
export function countableLength(text: string): number {
  // 공백 제외, 한글 글자 + 문장부호만 카운트
  return [...text].filter(ch => !/\s/.test(ch)).length
}
export function cpmFor(mult: number): number { return 400 * mult }
export function msPerChar(mult: number): number { return 60000 / cpmFor(mult) }
```
- **속도 변경 시:** `msPerChar`가 바뀌므로 스윕 바 타임라인(4-7)을 **전체 재빌드**한다.

### 4-7. 세로 스윕 바 — 타임라인·위치 계산·문장끝 멈춤·줄/페이지 전환

**핵심 아이디어:** "시간 축"과 "픽셀 축"을 분리해 한 줄을 **세그먼트(글자 1개 = 1세그먼트)** 타임라인으로 만든다. 각 세그먼트는 진행 시간(`msPerChar`)과 화면 위치(`startX→endX`)를 갖고, 문장 종결부면 끝에서 `holdMs=300` 만큼 멈춘다. **공백은 시간 0**이되 그 픽셀 폭은 **다음 글자 세그먼트의 시작 x로 흡수**시켜 순간이동(텔레포트)을 막는다.

- **입력:** `chars[]`(줄의 글자), `glyphX[]`(각 글자 우측 끝의 픽셀 x, Frontend가 글꼴 로드 후 측정해 주입), `msPerChar`(4-6).
- **출력:** `Segment[]`, 그리고 경과 시간 → x 위치 함수 `sweepXAt`, 한 줄 총시간 `lineDuration`.

```ts
// core/sweep.ts
export interface Segment { startMs:number; endMs:number; startX:number; endX:number; holdMs:number }
const SENTENCE_END = /[.?!…]/   // 문장 종결부 → 0.3초 멈춤

export function buildLineTimeline(chars:string[], glyphX:number[], msPerChar:number):Segment[] {
  const segs:Segment[] = []
  let t = 0, prevX = 0
  for (let i = 0; i < chars.length; i++) {
    const ch = chars[i], xEnd = glyphX[i]
    if (/\s/.test(ch)) { prevX = xEnd; continue }       // 공백: 시간0, 폭은 다음 글자가 흡수
    const hold = SENTENCE_END.test(ch) ? 300 : 0
    segs.push({ startMs:t, endMs:t+msPerChar, startX:prevX, endX:xEnd, holdMs:hold })
    t += msPerChar + hold
    prevX = xEnd
  }
  return segs
}

export function lineDuration(tl:Segment[]):number {
  if (!tl.length) return 0
  const last = tl[tl.length-1]
  return last.endMs + last.holdMs
}

export function sweepXAt(tl:Segment[], elapsed:number):number {
  if (!tl.length) return 0
  for (const s of tl) {
    const holdEnd = s.endMs + s.holdMs
    if (elapsed <= s.endMs) {                            // 이동 구간: 선형 보간
      const r = (elapsed - s.startMs) / (s.endMs - s.startMs)
      return s.startX + (s.endX - s.startX) * Math.max(0, Math.min(1, r))
    }
    if (elapsed <= holdEnd) return s.endX                // 문장 끝 0.3초 정지: x 고정
  }
  return tl[tl.length-1].endX
}
```

- **렌더링 루프(Frontend):** `requestAnimationFrame`마다 "현재 줄 경과 ms"를 구해 `sweepXAt`로 막대 x를 갱신한다.
- **줄/페이지 전환(의사코드):**
```
경과ms ≥ lineDuration(현재줄):
    경과ms -= lineDuration(현재줄)
    줄인덱스 += 1
    줄인덱스 > 마지막줄?  →  페이지 += 1, 줄인덱스 = 0
        페이지 > 마지막페이지?  →  (글 끝) 시간 남으면 처음부터(D4), 아니면 종료
    새 줄의 glyphX로 타임라인 재빌드
```

### 4-8. 일시정지 / 재개 — 상태 보존

- **보존 상태:** `{ pageIndex, lineIndex, lineElapsedMs, remainingTimerMs, charsRead }`.
- **일시정지:** 스윕 바 애니메이션 정지 + 타이머 누적 중단(활성 시간 멈춤). 화면에 "잠깐 멈춤".
- **재개:** 저장된 `lineElapsedMs`부터 `sweepXAt` 재개, `remainingTimerMs`부터 타이머 재가동. 위치·남은 시간·진도가 끊김 없이 이어진다.
- 휴식(4-9)도 같은 메커니즘으로 정지/복원한다.

### 4-9. 휴식 + 명언

- **트리거:** **활성 읽기 시간 5분(300000ms) 누적마다** 휴식. 휴식 중에는 활성 시간·타이머·스윕 바 모두 정지.
- **길이·스킵 불가:** `BREAK_MS = 20000`(20초), **건너뛰기 버튼 없음.** 20초 카운트다운을 다 채운 뒤에만 복귀.
- **명언 선택(중복 방지):** 1000개에서 무작위로 뽑되 **최근 표시한 N개는 제외**. 결정론적 테스트를 위해 난수기를 주입받는다.
```ts
// core/quotes.ts
export interface Quote { id:number; body:string; source:string; license:string }
export function pickQuote(all:Quote[], recentIds:number[], rng:()=>number = Math.random):Quote {
  const pool = all.filter(q => !recentIds.includes(q.id))
  const list = pool.length ? pool : all          // 모두 최근이면 전체에서 허용
  return list[Math.floor(rng() * list.length)]
}
```
- **데이터셋 기준(D6):** 총 1000개, **퍼블릭 도메인(속담·고전) 위주** 수집 → **10세 눈높이의 쉬운 우리말로 다듬어** 저장(어려운 한자어·관념어 지양). 부족분은 속담 각색·자체 창작으로 보충. `source`·`license` 컬럼 필수.
- **형식(JSON 시드):**
```json
[{ "id": 1, "body": "오늘 배운 작은 것이 내일의 큰 힘이 됩니다.", "source": "속담(다듬음)", "license": "public-domain" }]
```
- **예시 5개(10세 눈높이로 다듬은 문장):**
  1. "오늘 배운 작은 것이 내일의 큰 힘이 됩니다."
  2. "천 리 길도 한 걸음부터 시작돼요."
  3. "모르는 건 부끄러운 게 아니에요. 안 물어보는 게 아쉬운 거예요."
  4. "넘어져도 다시 일어나면, 그게 바로 자라는 거예요."
  5. "책 한 권에는 새로운 세상이 들어 있어요."

### 4-10. 진도·기록 저장 — 무엇을, 언제

- **세션 시작 시:** `sessions`에 행 생성(프로필·글·시작시각·사용 설정 `settings_json`).
- **진행 중:** 읽은 글자 수(`chars_read`)·도달 페이지(`page_reached`)를 메모리에서 갱신. (잦은 디스크 쓰기 회피.)
- **일시정지/휴식:** 활성 시간만 멈추고 진도 값은 유지(저장 시점은 종료 때).
- **세션 종료 시(타이머 끝):** `ended_at`·`active_ms`·`chars_read`·`page_reached`를 확정 저장. 명언을 보여줄 때마다 `quote_history`에 `(profile_id, quote_id, shown_at)` 기록(중복 방지용).

## 5. 데이터 구조 & DB

데이터는 두 층으로 나뉜다. **(A) 런타임 상태** — 읽는 동안 메모리에만 두는 값(자주 바뀜). **(B) 영구 저장** — SQLite에 두는 값(프로필·글·설정·기록·명언). 잦은 디스크 쓰기를 피하려고, 진행 중 값은 메모리에 두고 **세션 종료 시점에 한 번** 확정 저장한다.

### 5-1. 런타임 상태 구조 (메모리)

```ts
// 읽기 세션 한 번의 진행 상태 (저장 X, 종료 시 요약만 DB로)
interface SessionState {
  profileId: number
  textId: number
  // 위치
  pageIndex: number          // 현재 페이지
  lineIndex: number          // 페이지 안 줄 번호
  lineElapsedMs: number      // 현재 줄에서 흐른 시간(일시정지/재개 기준점)
  // 시간 (활성 시간 기준, D3)
  totalTimerMs: number       // 선택한 타이머 (예: 10분 = 600000)
  remainingTimerMs: number   // 남은 시간 (pause/break 중 정지)
  activeMsSinceBreak: number  // 마지막 휴식 이후 누적 활성 시간 (5분 = 휴식 트리거)
  // 진도
  charsRead: number          // 공백 제외 읽은 글자 수
  // 상태 플래그
  phase: 'reading' | 'paused' | 'break' | 'ended'
  breakRemainingMs?: number  // 휴식 중 남은 카운트다운 (20초→0)

// 화면 표시용 (글꼴 측정 결과 포함)
interface PageLayout {
  pages: string[][]          // 페이지 → 줄 → 문자열

```

핵심: `SessionState`는 4-7·4-8의 계산 입력이자, 일시정지/휴식 시 그대로 보존했다가 복원되는 단일 출처(single source of truth)다.

### 5-2. SQLite 테이블 스키마 (마이그레이션 0001)

better-sqlite3(동기 SQLite)를 사용하고, 앱 최초 실행 시 아래 스키마를 생성한다. 외래키 제약을 켜고(`PRAGMA foreign_keys = ON`), 프로필 삭제 시 연관 데이터가 함께 지워지도록 `ON DELETE CASCADE`를 건다.

```sql
-- 프로필 (별명 + 아바타만, 실명 비수집 — D10)
CREATE TABLE profiles (
  id         INTEGER PRIMARY KEY,
  name       TEXT NOT NULL,            -- 별명
  avatar     TEXT,                     -- 아바타 식별자(아이콘 키)
  created_at TEXT
);

-- 프로필별 설정 (1:1)
CREATE TABLE settings (
  profile_id     INTEGER PRIMARY KEY,
  theme          TEXT     DEFAULT 'light',   -- 'light' | 'dark'
  font_pt        INTEGER  DEFAULT 24,        -- 글자 크기(24pt↑)
  lines_per_page INTEGER  DEFAULT 5,         -- 3 | 4 | 5
  speed_mult     REAL     DEFAULT 1.0,       -- 0.5 | 1.0 | 1.5 | 2.0
  timer_min      INTEGER  DEFAULT 10,        -- 5 | 10 | 15 | 20
  FOREIGN KEY(profile_id) REFERENCES profiles(id) ON DELETE CASCADE
);

-- 프로필별 글 (1:N) — 붙여넣기/.txt 모두 정규화된 body로 저장
CREATE TABLE texts (
  id         INTEGER PRIMARY KEY,
  profile_id INTEGER NOT NULL,
  title      TEXT,
  body       TEXT NOT NULL,            -- 줄바꿈 정규화된 본문(\n)
  created_at TEXT,
  FOREIGN KEY(profile_id) REFERENCES profiles(id) ON DELETE CASCADE
);

-- 세션 기록 (1:N) — 종료 시 1회 확정 저장
CREATE TABLE sessions (
  id            INTEGER PRIMARY KEY,
  profile_id    INTEGER NOT NULL,
  text_id       INTEGER,
  started_at    TEXT,
  ended_at      TEXT,
  active_ms     INTEGER,               -- 실제 읽은 활성 시간
  chars_read    INTEGER,               -- 공백 제외 읽은 글자 수
  page_reached  INTEGER,               -- 도달 페이지
  settings_json TEXT,                  -- 그 세션에 쓴 설정 스냅샷
  FOREIGN KEY(profile_id) REFERENCES profiles(id) ON DELETE CASCADE,
  FOREIGN KEY(text_id)    REFERENCES texts(id)    ON DELETE SET NULL
);

-- 명언 본문 (전역 공유, 1000개 시드)
CREATE TABLE quotes (
  id      INTEGER PRIMARY KEY,
  body    TEXT NOT NULL,               -- 10세 눈높이로 다듬은 문장
  source  TEXT,                        -- 출처(예: '속담(다듬음)')
  license TEXT                         -- 'public-domain' 등
);

-- 프로필별 명언 표시 이력 (중복 방지용)
CREATE TABLE quote_history (
  profile_id INTEGER NOT NULL,
  quote_id   INTEGER NOT NULL,
  shown_at   TEXT,
  FOREIGN KEY(profile_id) REFERENCES profiles(id) ON DELETE CASCADE,
  FOREIGN KEY(quote_id)   REFERENCES quotes(id)   ON DELETE CASCADE
);
```

### 5-3. 주요 쿼리 패턴

- **설정 복원:** `SELECT * FROM settings WHERE profile_id = ?` (없으면 기본값 INSERT).
- **글 목록:** `SELECT id, title, created_at FROM texts WHERE profile_id = ? ORDER BY created_at DESC`.
- **명언 뽑기(중복 방지):** 최근 N개 id 조회 →
  `SELECT quote_id FROM quote_history WHERE profile_id = ? ORDER BY shown_at DESC LIMIT N` → 그 목록을 `pickQuote`의 `recentIds`로 전달 → 선택 후 `quote_history`에 INSERT.
- **세션 저장:** 종료 시 `UPDATE sessions SET ended_at=?, active_ms=?, chars_read=?, page_reached=? WHERE id=?`.

### 5-4. 시드 & 마이그레이션

- **마이그레이션:** `db.ts`가 `user_version`(PRAGMA)으로 버전을 확인해 0001 스키마를 적용. 이후 변경은 0002, 0003…로 누적.
- **명언 시드:** `data/quotes.seed.json`(1000개)을 최초 실행 시 `quotes` 테이블에 적재. 이미 있으면 건너뜀(중복 적재 방지).

## 6. 아동 안전 · 접근성 · 눈 피로 배려

이 앱은 어린이가 직접 쓰고 **눈을 오래 쓰는** 도구이므로, 안전과 눈 보호를 기능의 일부로 설계한다.

### 6-1. 눈 피로 줄이기
- **강제 휴식(스킵 불가):** 활성 읽기 5분마다 20초 휴식(4-9). 아이가 멈추기 싫어도 시스템이 쉬게 한다 — 눈의 깜빡임 회복과 초점 이완.
- **다크모드:** 어두운 환경에서 눈부심을 줄이는 다크 테마 제공. 스윕 바도 자극이 적은 따뜻한 앰버색.
- **큰 글자:** 기본 24pt 이상, −/+로 더 키우기 가능. 받침이 또렷한 Pretendard로 글자 변별을 쉽게.
- **반투명·부드러운 이동:** 스윕 바는 투명도 50%로 글자를 가리지 않고, 문장 끝에서 0.3초 멈춰 시선을 안정시킨다(급격한 추적 줄임).

### 6-2. 조작 안전·단순함 (아동 UX)
- **큰 버튼·적은 선택지:** 화면마다 핵심 동작만 큼직하게. 옵션은 3~4개 토글로 제한.
- **되돌리기 쉬움:** 일시정지/재개로 언제든 멈출 수 있고, 종료 화면에서 다시 시작·프로필 변경이 명확.
- **실수 방지:** 프로필/글 삭제 같은 파괴적 동작은 부모용 확인 1회를 둔다.

### 6-3. 개인정보·보호자 안내 (D10)
- **최소 수집:** **실명·생년월일을 받지 않고 별명+아바타로만** 아이를 구분.
- **완전 로컬:** 모든 데이터는 이 컴퓨터의 로컬 SQLite에만 저장되고 **인터넷으로 전송되지 않는다.** 프로필 화면·종료 화면에 이 사실을 한 줄로 안내해 부모를 안심시킨다.
- **자동 업데이트 없음:** 외부 통신을 만들지 않아 표면적(공격·추적 면)을 줄인다.

### 6-4. 접근성
- 충분한 명도 대비(라이트/다크 모두), 큰 터치·클릭 영역, 키보드로도 주요 동작(시작·일시정지) 가능하도록 설계.

## 7. Electron 구성 개요

데스크톱 앱은 Electron의 **3개 영역**으로 나눠 책임을 분리한다. 보안을 위해 렌더러(화면)에서 Node를 직접 못 쓰게 하고(`nodeIntegration: false`, `contextIsolation: true`), 안전한 통로(preload)로만 기능을 노출한다.

### 7-1. 역할 분리
- **main(메인 프로세스):** 앱 부팅, 창(`BrowserWindow`) 관리, **SQLite 접근(better-sqlite3)**, **.txt 파일 열기**(`dialog.showOpenDialog`), 종료 알림. 모든 디스크·DB 작업은 여기서만.
- **preload:** `contextBridge.exposeInMainWorld('api', …)`로 화면이 쓸 수 있는 **안전한 함수 묶음(`api`)**만 노출. 화면은 이 `api`를 통해서만 main에 요청한다.
- **renderer(화면):** UI 5개 화면 + **읽기 엔진**(`core/` 소비 + 글꼴 측정 + `requestAnimationFrame` 루프). `core/`의 순수 함수로 계산하고, 결과를 화면에 그린다.

### 7-2. IPC 계약(발췌)
```ts
// src/shared/ipc-contract.ts — main↔renderer 공유 타입
export interface Api {
  profiles: { list(): Promise<Profile[]>; create(name:string, avatar:string): Promise<Profile>; remove(id:number): Promise<void> }
  texts:    { list(profileId:number): Promise<TextItem[]>; save(profileId:number, title:string, body:string): Promise<TextItem>;
              importTxt(): Promise<{title:string; body:string} | null> }   // main의 파일 대화상자 사용
  settings: { get(profileId:number): Promise<Settings>; set(profileId:number, s:Settings): Promise<void> }
  session:  { save(s:SessionRecord): Promise<void> }
  quotes:   { next(profileId:number): Promise<Quote> }   // 최근 제외 후 무작위 + history 기록
}
```

### 7-3. 단일 .exe 패키징 (개념)
- **electron-builder + NSIS**로 **단일 설치 .exe**를 만든다(자동 업데이트 없음). 앱 아이콘·설치 경로·바탕화면 아이콘 포함.
- **네이티브 모듈(D8):** better-sqlite3는 Electron 버전에 맞춰 ABI **재빌드**(`electron-rebuild`)가 필요. **Phase 1에서 미리 검증**해 패키징(Phase 8) 단계의 폭탄을 방지.
- **글꼴 동봉(D5):** Pretendard(OFL)를 앱 리소스에 포함해 어느 PC에서나 동일하게 보이게 한다.
- **빌드 환경(D9):** 현재 Windows 환경에서 직접 빌드·설치·스모크 테스트.

## 8. 향후 확장 아이디어

지금 범위(MVP) 밖이지만, 같은 데이터 구조 위에서 자연스럽게 덧붙일 수 있는 후보들이다.

1. **난이도 단계:** 글 길이·속도·줄 수를 묶은 "초급/중급/고급" 프리셋으로 아이가 단계별로 도전.
2. **프로필별 읽기 리포트:** `sessions` 데이터를 모아 "이번 주 읽은 시간·글자 수·평균 속도"를 부모용으로 요약.
3. **통계 그래프:** 날짜별 읽기 시간·속도 변화를 간단한 막대/선 그래프로 시각화(성장 체감).
4. **명언 즐겨찾기:** 아이가 마음에 든 명언을 별표 저장해 모아 보기.
5. **글 카테고리/추천:** 동화·과학·역사 등 분류와 연령 맞춤 추천 글 제공.
6. **읽기 정확도 측정(선택):** 읽은 뒤 간단한 1~2문항 이해 퀴즈로 문해력 변화 점검(아동 부담 최소화 전제).
7. **소리 보조:** 페이지 전환·휴식 시작에 부드러운 효과음(켜고 끄기 가능).
