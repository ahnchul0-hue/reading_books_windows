당신은 교육용 데스크톱 앱을 설계하는 시니어 소프트웨어 아키텍트입니다.
아래 CO-STAR 규격에 따라, "10살 아이의 문해력·시선처리 훈련용 Windows 프로그램"의
**설계 명세서(Design Spec)**를 작성하세요. (※ 실제 코드 구현은 하지 말 것. 이 단계는 설계까지입니다.)

# CONTEXT (배경)
- 만들 대상: 줄글을 왼쪽→오른쪽으로 시선을 끊지 않고 지속적으로 읽도록 유도하는 읽기 훈련 앱.
- 사용자: 만 10세 전후 아동(직접 조작) + 옆에서 도와주는 부모. **여러 아이를 프로필로 구분해 사용**.
- 목적: 아이의 문해력 향상과 시선 처리(좌→우 안구 운동) 훈련.
- 실행 환경: Windows 데스크톱. 기술 스택은 **Electron** 데스크톱 앱으로 전제하고,
  배포는 **단일 .exe 설치본**(자동 업데이트는 불필요)으로 한다.
- 핵심 기능(반드시 모두 설계에 포함):
  1) 사용자 프로필: 여러 아이를 **프로필별로 등록/선택**하고, 기록·설정·읽던 글을 프로필 단위로 분리 저장.
  2) 텍스트 입력: 읽을 글을 ① **직접 붙여넣기(복사·붙여넣기)** 또는
     ② **.txt 파일 가져오기(열기)** 두 방식 모두로 넣을 수 있어야 한다.
     - 입력한 글은 **프로필에 저장**해 다음에 목록에서 다시 골라 읽을 수 있어야 한다.
  3) 페이지 표시: 글을 한 화면에 **3줄 / 4줄 / 5줄 중 선택**해 페이지 단위로 보여준다.
     한 페이지를 다 읽으면 다음 페이지로 전환한다.
     - 글꼴: **받침이 잘 보이는 고딕 계열**, 기본 **24pt 이상**,
       화면의 **−/+ 버튼으로 글자 크기 확대·축소** 가능(설정은 프로필별로 기억).
  4) 테마(라이트/다크 선택 가능):
     - 라이트 모드: 스윕 바 = **연한 노랑**, 글자폭 **1.5배**, 투명도 **50%**.
     - 다크 모드: 스윕 바 = **따뜻한 연한 호박색(앰버, 예: #F0C674 계열)**, 글자폭 **1.5배**, 투명도 **50%**.
     - 테마 선택은 프로필별로 기억.
  5) 타이머: 5/10/15/20분 중 선택(5분 단위)해 세션 실행. 종료 시 알림.
  6) 속도 선택: 0.5배속 / 1.0배속 / 1.5배속 / 2.0배속 중 선택.
     **기준 속도(1.0배속) = 분당 400자(400 CPM)**.
     0.5배=200CPM, 1.5배=600CPM, 2.0배=800CPM.
     **글자 수 카운트 규칙: 공백(띄어쓰기·줄바꿈)은 제외하고, 한글 글자와 문장부호만 1자로 센다.**
  7) 시선 유도(핵심): **반투명 세로 막대(스윕 바)**가 위 속도(CPM)에 맞춰
     글 위를 왼쪽→오른쪽으로 일정하게 이동하며, 아이의 시선이 막대를 따라가게 한다.
     줄 끝에 도달하면 다음 줄 시작점으로 내려가고, 페이지 끝이면 다음 페이지로 넘어간다.
     **문장 끝(마침표·물음표·느낌표 등 문장 종결부)에서는 0.3초간 멈춰** 시선을 안정시킨 뒤 이어간다.
  8) 일시정지·재개: 읽는 도중 **일시정지(Pause) 버튼**으로 스윕 바와 타이머를 함께 멈추고,
     **재개(Resume)** 시 멈춘 지점·남은 시간부터 이어서 진행한다.
  9) 휴식 + 명언:
     - **세션 진행 중 5분마다, 20초간 휴식 화면**을 자동으로 띄운다(스윕 바·타이머 일시정지).
     - **휴식은 건너뛸 수 없다(스킵 불가).** 20초를 모두 채운 뒤에만 읽기로 돌아간다(남은 시간 카운트다운 표시).
     - 휴식 화면에는 **공부·배움·성장에 관한 짧은 명언**을 하나 크게 보여준다.
       명언은 **총 1000개 문장 데이터셋**에서 무작위로 뽑되, 최근 표시한 것은 잠시 제외(중복 방지).
     - 명언 데이터셋 기준:
       · **출처는 자유 이용(퍼블릭 도메인) 위주**로 수집한다.
       · **모든 명언은 10세가 읽기 쉽도록 쉬운 우리말로 다듬어** 저장한다(어려운 한자어·관념어 지양).
       · 형식(JSON 또는 SQLite 테이블), 중복 방지(최근 표시 제외) 방안을 설계에 포함.
       · (※ 설계 단계이므로 1000개 전체를 나열하지 말고, 데이터 구조·수집/다듬기 기준·예시 5개만 제시.)
  10) 세션 종료 조건: **선택한 타이머 시간이 끝나면 세션을 종료**한다(타이머 우선).
      글을 끝까지 다 읽어도 타이머가 남아 있으면, 글을 처음부터 다시 돌리거나 대기하는 방식을 명시.
  11) 기록 저장: 세션마다 읽은 분량·진도·설정을 **SQLite**에 **프로필별로** 간단히 저장한다.

# OBJECTIVE (목표 산출물)
- 개발자에게 그대로 넘겨 구현을 시작할 수 있는 **설계 명세서**.
- 코드는 작성하지 않는다. 단, 핵심 로직·데이터 모델·DB 스키마는
  "의사코드(pseudocode)"나 스키마 형태로만 표현 가능.

# STYLE (문체)

- 핵심 구조·의사코드·데이터 모델·DB 스키마는 코드블록으로 깔끔히 제시.

# TONE (톤)
